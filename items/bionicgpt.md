# Bionic

## 一句话总结

一个支持本地或数据中心部署的企业级私有生成式 AI 平台，能为团队提供受控聊天、RAG 助手、权限治理和 API 化能力。

## 它解决什么问题

- 很多企业想用生成式 AI，但不愿把内部数据和使用日志交给外部服务
- 团队不仅需要聊天入口，还需要权限、团队隔离、审计和安全控制
- 想做企业内部 AI 助手时，经常缺少可控的数据治理和 API 暴露能力

## 适合谁

- 对数据保密要求较高的企业内部效率团队
- 需要自托管 AI 平台的外贸、金融、制造和互联网组织
- 想统一管理团队助手、模型访问和知识数据集的技术与运营团队

## 核心能力

- 私有化部署：支持在本地、数据中心或 Kubernetes 环境运行
- 团队级助手平台：支持助手共享、团队隔离、RBAC 和 SSO
- 企业级治理：支持审计、限流、数据集管理、Guardrails 和 API key 管理

## 典型使用场景

- 在公司内网搭建替代公共 ChatGPT 的团队 AI 工作台
- 按团队隔离知识库和助手，避免不同部门数据串用
- 把内部助手封装成 API 给客服、运营或分析系统调用

## 为什么值得关注

- 它不是单纯的“本地聊天界面”，而是比较完整的企业 AI 基础设施
- 对权限、审计、数据治理和安全的强调很适合正式业务场景
- 可同时覆盖内部员工助手和 API 输出两个方向
