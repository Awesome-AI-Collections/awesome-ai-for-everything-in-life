# Apollo.io MCP Server

## 一句话总结

把 Apollo.io 销售情报能力封装成 MCP server，方便 agent 工作流调用获客与 enrichment 能力。

## 它解决什么问题

- 开发者想把 Apollo.io 接进 agent，但不想手写大量 API 封装
- 销售自动化工作流常常需要统一的 lead search、enrichment 和 sequence 调用层
- 团队做 AI SDR 或销售助手时，希望直接复用 MCP 工具接口

## 核心场景

- AI 销售代理接入 Apollo.io 数据
- MCP 工具链中的 lead generation 能力封装
- 销售自动化 agent 的工程化搭建

## 为什么值得关注

- 它不是单独业务应用，而是可复用的销售数据能力接口层
- 对“agent + 销售情报”工作流很有实战价值
- 很适合放进企业内部的销售自动化工具箱
