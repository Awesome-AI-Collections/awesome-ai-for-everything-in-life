# OpenAdapt

## 一句话总结

一个把桌面和网页操作录制、学习、评测和执行串起来的开源 AI 流程自动化工具，适合运营、财务和后台团队把重复 GUI 操作逐步自动化。

## 它解决什么问题

- 很多后台岗位每天要在 ERP、表单、浏览器后台和桌面工具里重复点选、复制和录入，传统 RPA 配置成本又很高。
- 纯脚本自动化往往要求接口或代码能力，但大量业务流程仍然只存在于 GUI 界面里。
- 团队想把真实人工操作转成可复用自动化流程时，缺少一个“先示范、再学习、再执行”的开源底座。

## 适合谁

- 做运营、财务、客服后台和数据录入的流程团队
- 需要把桌面和网页混合工作流自动化的内部工具团队
- 想探索 AI-first RPA 或数字员工方案的企业

## 核心能力

- 操作录制：可记录人类在 GUI 中的真实操作过程，沉淀成演示轨迹。
- 学习与评测：支持基于演示进行训练、检索和评测，逐步提升自动化质量。
- 执行闭环：把观察、决策、定位和执行串起来，形成可运行的自动化代理。

## 典型使用场景

- 运营团队自动化处理后台审批、表单录入和重复网页操作。
- 财务或行政团队把跨多个系统的固定流程录制下来，逐步交给 AI 执行。
- 内部自动化团队用它验证“示范驱动”的 GUI 自动化方案是否可落地。

## 为什么值得关注

- 它不是简单脚本录制器，而是把演示、检索、训练和执行做成了一条完整链路。
- 对没有 API、只能靠界面操作的老系统场景尤其有价值。
- 近期 release 仍在更新，说明项目还在往更可用的自动化底座推进。

## 类似项目

- [UiPath](https://www.uipath.com) - 更成熟的商业 RPA 路线，可用来对比开源与企业版自动化方案。
- [Zapier](https://zapier.com) - 更偏 API 与 SaaS 集成自动化，不擅长 GUI 级演示学习。
