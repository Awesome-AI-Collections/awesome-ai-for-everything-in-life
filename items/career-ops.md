# Career-Ops

## 一句话总结

一个基于 Claude Code 的 AI 求职系统，把职位扫描、岗位评估、定制简历、批量处理、投递跟踪和面试准备串成一条完整求职流水线。

## 它解决什么问题

- 手工投简历很容易陷入“海投”，却缺少稳定的岗位筛选标准
- 针对不同岗位反复改简历、做调研、准备面试材料，时间成本很高
- 求职过程常分散在表格、文档、浏览器和笔记里，难以形成单一真相源

## 适合谁

- 想系统化管理求职流程的候选人
- 需要针对不同 JD 快速定制简历和材料的人
- 希望把职位发现、评估、跟踪和面试准备交给 AI 辅助的人

## 核心能力

- 求职流水线：从职位链接输入开始，自动进入评估、报告、PDF 和 tracker 流程
- 岗位筛选：用结构化评分而不是关键词匹配来判断岗位适配度
- 简历定制：按具体 JD 生成更适合 ATS 的 PDF 简历
- 投递中台：把职位、状态、历史和待处理项集中管理
- Agent 化协作：支持 Playwright 扫描招聘门户，也支持并行批处理多个岗位

## 模式拆解

### auto-pipeline

- 作用：输入职位链接或描述后，自动完成评估、PDF 生成和 tracker 记录
- 价值：把原本分散的求职动作收敛成一次完整处理

### scan

- 作用：扫描 Ashby、Greenhouse、Lever、Wellfound 等招聘门户和预设公司列表
- 价值：适合定期找新岗位，减少手工刷新职位页

### oferta / evaluate

- 作用：对单个岗位做多维度匹配评估，包括角色理解、能力匹配和差距分析
- 价值：帮助求职者优先投入到更值得申请的职位

### pdf

- 作用：根据目标职位生成 ATS 优化的 PDF 简历
- 价值：降低为不同岗位重复改简历的成本

### batch

- 作用：并行处理多条岗位信息，批量做评估
- 价值：适合在一轮集中筛选中快速缩小目标范围

### tracker / pipeline

- 作用：维护职位状态、待处理链接和应用进展
- 价值：把求职过程从零散表格提升为可追踪的流程系统

### apply

- 作用：辅助填写申请表单，但不自动替用户提交
- 价值：保留人工把关，同时减少重复输入

### deep / interview-prep / contacto

- 作用：补充公司深度研究、面试准备和 LinkedIn 外联消息生成
- 价值：让这个系统不只服务“投递前”，也覆盖后续沟通与面试阶段

## 典型使用场景

- 定期扫描目标公司招聘页，先筛出高匹配岗位再决定是否申请
- 针对单个岗位快速生成定制简历、评估报告和投递记录
- 一次性批量处理十几个职位，再按得分排序安排申请优先级
- 面试前汇总公司研究、行为面试故事和谈薪话术

## 为什么值得关注

- 它不是简单“帮你投简历”，而是把求职当成一套可持续优化的运营系统
- README、AGENTS 和 CLAUDE 文档把数据边界、个性化位置和工作流说得比较完整
- 强调 human-in-the-loop，不主张无脑自动海投，这点比很多自动投递工具更克制
