# GEOFlow

## 一句话总结

一个把 AI 生成、素材管理、任务调度、草稿审核和前台发布串成完整链路的 GEO / SEO 内容生产系统，适合搭建自动化内容站点或内部内容运营后台。

## 快速判断

- 如果你想把 GEO / SEO 内容生产做成“配置模型 -> 跑任务 -> 审核 -> 发布”的后台系统，它值得优先看
- 如果你只需要零散 prompt 或轻量文章生成，不一定需要它这么完整的内容运营链路
- 它更像内容运营工作台和生产系统，不是单纯 skill 或 CLI 工具

## 你会怎么用它

- 接进 AI 开发流：更适合作为运营侧内容生产后端，被 skill、CLI 或自动化脚本调用，而不是作为 coding agent 产品本体
- 接进日常工作流：把标题库、关键词库、图片库、知识库、任务调度和审核发布统一收进一个运营后台
- 最小落地方式：先在一个小型站点或内部内容项目里跑通“任务 -> 草稿 -> 审核 -> 发布”链路，再决定是否深度接入

## 它解决什么问题

- 内容运营经常卡在素材分散、任务零散、审核发布不成体系
- 只会调用模型生成文章，不等于真正具备可运营、可调度、可审核的内容生产系统
- 做 GEO / SEO 站点时，往往既要批量生成，又要控制素材、质量和发布节奏

## 适合谁

- 想用 AI 批量生产 GEO / SEO 内容的运营团队
- 需要内容任务调度、审核和发布链路一体化的增长团队
- 想搭建自动化内容站点或内部内容运营后台的个人与组织

## 核心能力

- 多模型内容生成：兼容 OpenAI 风格接口，可接不同 AI 服务商
- 批量任务调度：支持定时、队列执行、失败重试和 worker 链路
- 素材统一管理：标题库、关键词库、图片库、知识库、提示词集中维护
- 审核与发布工作流：草稿、审核、发布三段式流程，可切换自动发布
- SEO / GEO 导向：支持元信息、Open Graph、结构化数据等搜索展示优化
- 可部署后台：支持 Docker Compose 和本地 PHP 方式快速部署

## 能力边界

- 明显可用：GEO / SEO 内容站点、内部内容运营后台、批量内容任务管理
- 效果一般：只做单篇文章实验、没有审核流程的小团队、完全不需要后台系统的场景
- 不要误用：不要把它当成单纯文章生成器；它的价值在完整内容生产链路，而不只是“调用模型写文”

## 集成方式

- 作为单独工具：直接部署成内容运营后台和前台站点
- 作为 AI 工作流组件：放在内容生产、审核、调度和发布层，被本地 CLI、skill 或外部脚本调用
- 作为团队流程节点：适合让运营、审核和发布协作围绕同一套任务和素材系统运转

## 上手建议

- 第一步先用 Docker 跑一个本地实例，把模型、提示词和素材库配置起来
- 最值得先试的是“重复性高、审核标准明确”的内容任务
- 如果你当前还不确定需求规模，先轻量试一条主题线，不要一开始就搭大而全的站群系统

## 选型建议

- 如果你的主需求是内容生产后台和运营链路一体化，适合看它
- 如果你的主需求只是营销 skill 或 prompt 资产，`AI Marketing Skills`、`dbskill` 一类工具更轻
- 如果你最在意的是任务调度、审核流程和发布闭环，GEOFlow 的价值会更明显

## 为什么值得关注

- 它把很多团队口中的“AI 内容生产”真正做成了可部署、可审核、可发布的系统
- 不只给生成按钮，而是把素材、任务、队列、审核和站点输出都串起来了
- 对想做 GEO / SEO 自动化站点的人来说，这类完整后台比单点写作工具更接近实际落地
