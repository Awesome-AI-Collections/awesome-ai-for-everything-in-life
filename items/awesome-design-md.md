# awesome-design-md

## 一句话总结

一个收集真实网站 DESIGN.md 模板的精选仓库，可直接为 AI 生成 UI 提供可复用的设计系统文本。

## 它解决什么问题

- 用 AI 生成页面时，最难的常常不是代码，而是稳定的视觉约束
- 设计规范常散落在 Figma、组件库和口头习惯里，不利于 Agent 读取
- 想复用成熟产品的风格时，缺少适合 LLM 理解的轻量模板

## 适合谁

- 用 AI 生成前端页面的设计师和开发者
- 想快速借鉴成熟产品视觉语言的独立开发者
- 需要把设计规则文本化、方便 Agent 消化的团队

## 核心能力

- DESIGN.md 模板库：收录多个真实网站风格的现成模板
- 结构清晰：覆盖配色、排版、组件、布局、响应式等要素
- 低门槛复用：复制一份 DESIGN.md 到项目里即可给 Agent 用
- 适配 AI 工作流：天然适合 Google Stitch 和各类前端 Agent 读取

## 典型使用场景

- 在项目根目录放入 DESIGN.md，让 Agent 按指定风格生成页面
- 借用 Stripe、Linear、Notion 等风格做快速设计探索
- 为团队沉淀一套更适合 AI 理解的设计系统表达方式

## 为什么值得关注

- 它抓住了 AI 生成 UI 的关键问题：给模型明确、可读的视觉边界
- 相比重度设计工具，Markdown 形式更轻、更适合 Agent
- 对需要快速生成一致性 UI 的个人和小团队很有参考价值
