# Remotion

## 一句话总结

一个用 React 代码来生成视频的框架，适合把模板、数据、动画和品牌元素做成可重复的视频生产系统。

## 它解决什么问题

- 许多视频生产流程依赖手工剪辑，不适合做大规模模板化和自动化复用。
- 当视频内容需要根据数据、变量或品牌规则动态生成时，传统剪辑工具不够灵活。
- 熟悉前端技术的团队想做程序化视频，但往往缺少成熟的 React 路线工具。

## 适合谁

- 想用代码驱动视频生产的创作者和设计工程师
- 需要批量生成营销、教育或数据视频的团队
- 已经熟悉 React，希望把视频做成组件化模板的人

## 核心能力

- React 视频编排：用组件、状态和组合方式来描述视频内容。
- 程序化表达：可直接利用 CSS、Canvas、SVG、WebGL、函数和算法生成效果。
- 模板化生产：适合把品牌组件、字幕、动画和数据输入沉淀成可复用模板。
- 完整文档与展示：官方同时提供 docs、API 和 showcase，便于学习和选型。

## 典型使用场景

- 批量生成带数据驱动图表和字幕的视频内容。
- 为品牌视频、产品演示或年终总结做程序化模板。
- 把视频生产嵌进内容工作流，而不是每次手工重复剪辑。

## 为什么值得关注

- 它把视频创作从“操作软件”切换成“写组件和模板”，可维护性更强。
- 对已经熟悉 React 的团队来说，上手成本比学习新的重型视频脚本系统更低。
- 在 `Design / Creative` 里，它代表了“代码即视频生产线”的一类重要工具。

## 类似项目

- [pixelle-video](pixelle-video.md) - 更偏一句话自动生成短视频，而 `Remotion` 更强调开发者自己定义视频逻辑与模板。
- [videolingo](videolingo.md) - 更偏现有视频的翻译与处理工作流，而 `Remotion` 更适合从零程序化生成视频。
