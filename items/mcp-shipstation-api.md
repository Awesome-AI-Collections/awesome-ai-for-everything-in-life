# ShipStation MCP Server

## 一句话总结

一个连接 ShipStation API v2 的 MCP server，可让 AI 直接处理 shipments、labels、rates、inventory 和 warehouses 等履约能力。

## 它解决什么问题

- 发货、打单、运费试算和仓库管理分散在 ShipStation 后台多个模块里。
- 团队想让 AI 辅助处理履约动作，但没有统一的接口层。
- 外贸和电商运营常常需要把物流动作接给 AI 或自动化流程。

## 适合谁

- 物流与履约团队
- 跨境电商发货团队
- 想把 ShipStation 接进 AI 助手的业务团队

## 核心能力

- 履约动作：支持 shipments、labels、batches 和 manifests。
- 运费能力：支持 rates 和 estimate rates。
- 运营数据：支持 carriers、inventory、warehouses 等对象。

## 典型使用场景

- 让 AI 辅助创建和管理发货标签。
- 做发货前的运费估算和承运商比较。
- 把履约动作接到客服、运营或仓库 AI 助手里。

## 为什么值得关注

- 这是官方 ShipStation 仓库，可信度高。
- 覆盖的履约能力很贴近实际业务操作。
- 对外贸、电商和互联网履约团队都很实用。

## 类似项目

- [eBay MCP](./ebay-mcp.md) - 更偏卖家后台与 listing，ShipStation 更偏履约发货。
- [CData JDBC MCP Server](./cdata-jdbc-mcp-server.md) - 更偏多平台数据查询，不负责 ShipStation 动作执行。
