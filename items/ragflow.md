# RAGFlow

## 一句话总结

一个开源 RAG 与 Agent 引擎，适合把公司文档、知识库和业务资料整理成可检索、可追溯的问答系统。

## 它解决什么问题

- 公司文档、制度、产品资料和项目知识分散在各处，员工查询成本高
- 传统关键词搜索很难处理长文档、表格、扫描件和复杂非结构化资料
- 团队想搭企业知识库问答，但又担心引用不准、切块差和结果不可追溯

## 适合谁

- 需要内部知识库问答的研究、运营和支持团队
- 想把说明书、制度、培训材料和业务文档接给 AI 的企业
- 需要更强文档理解和引用追踪能力的团队

## 核心能力

- 深度文档理解：支持复杂文档、扫描件、网页和多类业务资料
- RAG 编排：支持切块、检索、重排和可追溯引用
- Agent 扩展：把知识问答和 agent 工作流结合起来

## 典型使用场景

- 搭建公司制度、产品手册和 SOP 的内部问答系统
- 让销售、客服或外贸团队快速检索资料并获得带引用答案
- 为研究和分析团队整理长文档、报表和知识库内容

## 为什么值得关注

- 它在文档理解和上下文质量上投入很多，不只是简单向量检索
- 兼顾知识库问答和 agent 工作流，适合后续扩展
- 对企业文档和复杂资料场景的适配度比较高

## 类似项目

- [Company Researcher](company-researcher.md) - 更偏公司情报整合，不是内部知识库引擎
- [Open WebUI](open-webui.md) - 可承接文档问答体验层，但 RAGFlow 更像底层知识引擎
