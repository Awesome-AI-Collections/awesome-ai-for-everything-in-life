# Contact Use

## 一句话总结

一个用浏览器 Agent 自动寻找并尝试联系目标个人或组织的外联工具，适合冷启动联系场景。

## 它解决什么问题

- 目标对象的联系方式分散在网页、社媒和公开资料里，很难快速找到
- 团队做外联前的人工信息搜集很耗时间
- 需要一个能主动探索页面并找联系方式的 Agent，而不是纯搜索框工具

## 适合谁

- 做销售开发和商务拓展的团队
- 需要高频找联系入口的研究或运营人员
- 想把 Browser Agent 用到真实外联场景的人

## 核心能力

- 浏览器 Agent：自动探索网页并寻找联系入口
- 联系方式发现：帮助团队更快定位可触达路径
- 外联前置流程：适合做销售或 BD 的前期准备

## 典型使用场景

- 冷启动联系潜在客户或合作方
- 找官网表单、邮箱、社媒和其它触达入口
- 给销售或 BD 团队做外联前情报收集

## 为什么值得关注

- 它不是泛用浏览器自动化，而是明确对准“联系目标对象”
- 目标很实用，贴合真实销售和商务流程
- 对外联型团队来说，前置效率提升空间很大
