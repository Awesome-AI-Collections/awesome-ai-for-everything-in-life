# LibreTranslate

## 一句话总结

一个可自托管的开源机器翻译 API，适合外贸、出海和多语内容团队搭建自己的翻译能力，不依赖 Google 或 Azure。

## 它解决什么问题

- 外贸和出海团队需要频繁处理中英及多语内容，但不想把核心内容完全交给闭源翻译服务
- 产品文案、客服话术、邮件和帮助中心翻译往往分散在不同工具里
- 团队希望把翻译能力接进自己的内部系统、网站或工作流

## 适合谁

- 做外贸邮件、产品文案和多语内容运营的团队
- 需要自托管翻译能力的出海业务和国际化团队
- 想把翻译 API 接进内部平台或自动化流程的组织

## 核心能力

- 自托管翻译：整套机器翻译能力可以部署在自己的服务器上
- API 友好：适合接入网站、表单、客服或内部业务系统
- 开源可控：基于开源翻译引擎，便于按业务需要继续扩展

## 典型使用场景

- 给外贸团队的邮件、商品介绍和 FAQ 提供内部翻译接口
- 给官网、帮助中心和活动页做多语言翻译工作流
- 作为企业私有翻译中台接入 ERP、CRM 或内容系统

## 为什么值得关注

- 对“可控的翻译能力”这件事非常直接，不是只提供一个演示页面
- 对需要数据控制和成本控制的团队更友好
- 和多语内容、国际化流程、出海运营都有天然结合点

## 类似项目

- [Tolgee](tolgee.md) - 更偏本地化协作平台，而不是纯翻译 API
- [ltengine.md](ltengine.md) - 同样走本地自托管翻译能力，但更强调 LibreTranslate 兼容引擎
