# Vexa

## 一句话总结

一个可自托管的开源会议机器人与转写 API 平台，适合会议记录、客服质检和销售通话数据沉淀。

## 它解决什么问题

- 团队想把会议机器人、实时转写和后续 AI 分析接到自己系统里，但 SaaS 成本高且不够可控。
- 客服、销售和客户成功团队希望把会议数据沉淀成结构化资产，而不只是临时纪要。
- 很多受监管或重隐私行业不愿把会议数据直接交给外部平台。

## 适合谁

- 需要会议机器人和转写能力的产品与运营团队
- 想做销售通话分析、客服质检和会议数据平台的团队
- 需要自托管会议基础设施的企业

## 核心能力

- 会议机器人 API：可加入 Google Meet、Teams、Zoom 等会议。
- 实时转写：支持按说话人输出转写结果，并可通过 API 或 WebSocket 获取。
- 自托管部署：适合把会议数据保留在自有基础设施中。

## 典型使用场景

- 销售团队自动采集客户会议并沉淀转写数据用于复盘。
- 客服团队把通话或会议记录接入质检与知识库流程。
- 企业产品把会议机器人能力嵌入自己的 AI 助手或协作产品里。

## 为什么值得关注

- 不只是会议摘要工具，而是更底层、更可集成的会议数据平台。
- 自托管属性对重隐私行业和企业客户很关键。
- 对想把会议场景做成产品能力的团队非常实用。
