# MCP WaliChat

## 一句话总结

一个把 WaliChat 的 WhatsApp 业务能力接到 Claude、ChatGPT、Cursor 等 AI 客户端里的 MCP 连接器。

## 它解决什么问题

- 很多团队已经有 WhatsApp 业务流，但缺少能直接让 AI 助手操作消息、群组和报表的接口层。
- 外贸销售、客服和跟单希望把“发消息、查聊天、跑活动、做报表”交给 AI 统一处理。
- 只拿到底层 WhatsApp API 还不够，真正落地时更需要对 MCP 客户端友好的动作型工具集合。

## 适合谁

- 做 WhatsApp 销售外联、客户维护和售后支持的外贸团队
- 想把 Claude、ChatGPT 或 OpenAI Responses API 接到业务消息流里的团队
- 需要把客户沟通、标签、群组和营销活动自动化的互联网运营团队

## 核心能力

- MCP 接入：可直接给 Claude Desktop、ChatGPT、Cursor、Windsurf、VS Code Copilot 等客户端调用。
- WhatsApp 业务动作：覆盖消息、聊天、群组、标签、活动、设备、报表等大量业务操作。
- 分析与自动化：支持会话分析、活动管理、客户沉默识别、批量广播和智能业务流程。

## 典型使用场景

- 让 AI 助手直接帮外贸员查聊天、发跟进消息、跑欢迎活动和看客户活跃度
- 给客服或运营做 WhatsApp 标签管理、报表导出和自动回复工作流
- 把 OpenAI Responses API 或 Claude Desktop 接到现有 WaliChat 账号上做业务自动化

## 为什么值得关注

- 它不是泛泛的“WhatsApp + AI”示例，而是把业务动作真正做成了 MCP 工具集。
- README 明确对齐 2025 年 3 月 26 日的 MCP HTTP streaming 规范，并直接支持 ChatGPT、Claude Desktop、Cursor、Windsurf 等现代客户端。
- 对用户最实用的一点是：消息、群组、活动、标签、分析这些业务动作已经预先封装好了。

## 类似项目

- [WPPConnect Server](wppconnect-server.md) - 更偏 WhatsApp 服务端能力，不是 MCP 客户端连接器。
- [Evolution API](evolution-api.md) - 更偏 API 编排和业务系统集成层，而不是 MCP 工具层。
