# Evolution API

## 一句话总结

一个把 WhatsApp、Chatwoot、Dify、OpenAI、Typebot 等接到一起的开源消息接入平台，适合外贸客服、销售触达和社媒私信自动化。

## 它解决什么问题

- 外贸和互联网团队常常把 WhatsApp、客服台、机器人平台和 AI 服务拆成多套系统，协作很乱。
- 业务想把 WhatsApp 接上 AI 助手时，往往卡在 API、会话事件、媒体、Webhook 和第三方集成层。
- 如果只有单一消息入口，就很难把客服、销售和运营流程统一起来。

## 适合谁

- 做 WhatsApp 客服、销售外联和跟单自动化的团队
- 想把 Dify、OpenAI 或 Chatwoot 接进业务消息流的团队
- 需要把消息事件推给工作流、队列或客服系统的互联网团队

## 核心能力

- WhatsApp 接入：支持基于 Baileys 的 WhatsApp API，也支持官方 Cloud API。
- AI 与客服集成：可对接 Dify、OpenAI、Chatwoot、Typebot 等服务。
- 事件分发：支持 webhook、Kafka、RabbitMQ、SQS、WebSocket 等消息流转方式。

## 典型使用场景

- 把外贸客户的 WhatsApp 询盘接到 AI 助手和人工客服混合流程里。
- 把销售私信、客服消息和自动回复统一接到 Chatwoot 或机器人平台。
- 为运营团队做消息触发、媒体处理和自动化回复中台。

## 为什么值得关注

- 它不只是一个 WhatsApp 接口，而是开始往“消息入口 + AI 编排层”发展。
- 对外贸销售、客服、社媒私信和自动化跟进非常贴近。
- 虽然最新 release 在 2025 年 12 月，但集成范围和项目成熟度都足够实用。

## 类似项目

- [WPPConnect Server](./wppconnect-server.md) - 更偏开箱即用的 WhatsApp API Server，不像 Evolution API 这样强调 Dify、Chatwoot 和多集成编排。
- [Company Researcher](./company-researcher.md) - 更偏销售前的公司调研，不负责消息渠道和客服自动化。
